## Response Format

<ResponseField name="latest_block_height" type="integer" required>
  The height of the most recently processed block on the NEAR blockchain.
</ResponseField>

<ResponseField name="latest_block_timestamp" type="string" required>
  ISO 8601 timestamp of when the latest block was created.
</ResponseField>

<ResponseField name="total_transactions" type="integer" required>
  Total number of transactions indexed by the API.
</ResponseField>

<ResponseField name="total_ft_transfers" type="integer" required>
  Total number of fungible token (FT) transfers recorded.
</ResponseField>

<ResponseField name="total_nft_transfers" type="integer" required>
  Total number of non-fungible token (NFT) transfers recorded.
</ResponseField>

### Success Response (200 OK)

```json
{
  "latest_block_height": 179321914,
  "latest_block_timestamp": "2026-01-01T20:57:29.641844Z",
  "total_transactions": 17966163,
  "total_ft_transfers": 12788521,
  "total_nft_transfers": 9491
}
```

<Check>
  A `200 OK` status code indicates the statistics were retrieved successfully.
</Check>

## Understanding the Metrics

<AccordionGroup>
  <Accordion title="Latest Block Height" icon="blocks">
    The block height represents the total number of blocks that have been added to the blockchain since genesis. This number continuously increases as new blocks are produced.
    
    **Update frequency:** Approximately every 1-2 seconds on NEAR Protocol.
  </Accordion>
  
  <Accordion title="Latest Block Timestamp" icon="clock">
    The exact time when the most recent block was created and added to the blockchain. This timestamp is in UTC and follows the ISO 8601 format.
    
    **Use case:** Verify how fresh the indexed data is by comparing this with the current time.
  </Accordion>
  
  <Accordion title="Total Transactions" icon="arrows-turn-right">
    Cumulative count of all transaction records indexed by the ShadowCorp API. This includes all transaction types on NEAR blockchain.
    
    **Note:** This metric only includes transactions that have been indexed by our system.
  </Accordion>
  
  <Accordion title="FT Transfers" icon="coins">
    Total number of fungible token transfer events, including transfers of tokens like USDT, wrapped NEAR, and other NEP-141 compliant tokens.
    
    **Growth rate:** Typically grows faster than NFT transfers due to DeFi and trading activity.
  </Accordion>
  
  <Accordion title="NFT Transfers" icon="image">
    Total count of non-fungible token (NEP-171) transfer events, including mints, sales, and transfers of digital collectibles.
    
    **Insight:** Compare this with FT transfers to understand the ratio of collectible vs. fungible token activity.
  </Accordion>
</AccordionGroup>

## Example: Building a Statistics Dashboard

Here's a complete example of fetching and displaying network statistics:

```javascript
class NetworkStatsMonitor {
  constructor(apiKey) {
    this.apiKey = apiKey;
    this.baseUrl = 'https://api.shadowcorp.dev';
  }

  async getStats() {
    try {
      const response = await fetch(`${this.baseUrl}/v1/stats`, {
        headers: {
          'X-API-Key': this.apiKey
        }
      });

      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
      }

      return await response.json();
    } catch (error) {
      console.error('Error fetching network stats:', error);
      throw error;
    }
  }

  async displayStats() {
```

## Error Responses

<Warning>
  Make sure to include your API key in the `X-API-Key` header, otherwise you'll receive a `401 Unauthorized` error.
</Warning>

### 401 Unauthorized

```json
{
  "error": "Missing or invalid API key",
  "code": 401
}
```

### 429 Rate Limit Exceeded

```json
{
  "error": "Rate limit exceeded",
  "code": 429,
  "retry_after": 60
}
```
